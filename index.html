Étape 1 : Installation

Avant de commencer, assurez-vous d'avoir Node.js installé sur votre machine. Node.js permet d'exécuter du JavaScript partout. Vous pouvez vérifier votre installation en ouvrant un terminal et en exécutant : 

node -v

Vous devriez voir s’afficher une version de Node.js. Si ce n’est pas le cas, téléchargez et installez Node.js depuis nodejs.org.

Instructions :

  1. Placez-vous sur votre bureau.
  2. Ouvrez un terminal ou l'invite de commandes (cmd).
  3. Lancez les commandes suivantes :

            mkdir Svelte
            cd Svelte
            npm create vite@latest tp-svelte --template svelte

Lorsque vous y serez invité, sélectionnez Svelte comme framework, puis choisissez JavaScript comme langage.


            cd tp-svelte
            npm install
            npm install jspdf
            code .

Tester l'application :

Pour lancer votre application en mode développement, utilisez la commande suivante :


            npm run dev

Une fois démarrée, ouvrez l'URL fournie dans votre navigateur pour voir votre application en action.

Étape 2 : Votre premier composant

Pour organiser vos composants, commencez par créer un dossier dédié. Ouvrez un terminal et exécutez les commandes suivantes : 


            mkdir components
            cd components

Le premier composant que nous allons créer sera un Header. Par convention, les noms des composants commencent par une majuscule. Créez ce fichier en exécutant : 


            echo. > src/components/Header.svelte

Structure d’un composant

Comme vu dans le cours, un composant Svelte est divisé en trois sections : 


                        <script>
                            // Logique du composant
                        </script>
                        
                        <!-- Votre code HTML -->
                        
                        <style>
                            /* Styles CSS */
                        </style>

Dans ce TP, nos composants devront afficher des informations transmises via une variable. Cependant, ces informations ne doivent pas être définies dans le composant lui-même. Le composant doit être prêt à les afficher lorsqu’il sera appelé (par exemple, dans App.svelte).

Créer et exporter une variable

Dans le fichier Header.svelte, ajoutez la logique suivante :


                        <script>
                            export let titre;
                        </script>

Ensuite, ajoutez le code HTML ci-dessous pour afficher cette variable :


                        <div>
                            <h1>{titre}</h1>
                        </div>

Avec cela, votre composant Header.svelte est prêt à être utilisé. Il affichera la valeur de la variable titre lorsqu'elle sera transmise depuis le composant parent.

Étape 3 : Appeler le composant.

Vous êtes dans App.svelte. Supprimez tout. Copiez la structure de base ci-dessous et lisez bien les commentaires :


            <script>
            import jsPDF from "jspdf";
            // ici, vous importerez tous vos composants
            
            // Espace pour les variables partagées pour vos composants et l’instance doc de la classe jsPDF ci-dessous
            const generatePDF = () => {
                const doc = new jsPDF();
                // ici, vous construirez les pages du pdf.
                doc.save("example.pdf");
            };
            </script>
            
            <!-- Votre Code HTML
                Votre Prévisualisation
                Vos composants pour la prévisualisation
                Votre bouton pour générer le PDF (qui est aussi un composant!) -->
            
            <style>
            /* Votre code css pour la fausse pré-visualisation */
            </style>

A présent, importez le composant comme suit :


            import Header from "./components/Header.svelte";

Vous pouvez maintenant appeler votre composant comme ceci en le mettant au niveau de votre code HTML :


            <Header/>

Étape 4 : Injecter la variable.

Cependant, il manque quelque chose, souvenez-vous, le composant Header doit afficher une chaîne de caractère à partir d’une variable. Grâce à la directive d'exportation « export », la variable titre est accessible en tant que prop (propriété) depuis un composant parent (ici App.svelte).

Dans l’espace pour les variables partagées, ajoutez une chaîne de caractère pour le titre comme suit :


            const titre = "Un titre d'article Wikipédia";

Cette constante nous servira également pour la construction du pdf dans la méthode const generatePDF = () => {.

Maintenant, vous pouvez « injecter » titre dans votre composant tel quel : 


            <Header {titre} />

Étape 5 : Générer le PDF :

Regardons le code suivant :

Il existe de nombreuses méthodes pour construire votre PDF comme :

  • doc.line(x1, y1, x2, y2) : Dessine une ligne entre deux points (x1, y1, x2, y2);
  • doc.rect(x, y, width, height, "F") : Dessine un rectangle;
  • doc.addImage("votreImage", "FORMAT_EN_MAJUSCULE", x, y, width, height) : Ajoute une image;
  • et ainsi de suite...

Pour le challenge, vous pourriez avoir besoin de consulter la documentation de JSPDF.

Étape 6 : Le challenge.

Pour ce challenge, vous avez tout en possession pour le réussir ! N'hésitez pas à revenir en arrière et à regarder le cours. Il vous est demandé de reproduire ce que vous venez de faire.

N’essayez pas d’attribuer de valeur aux variables de vos composants !

Ci-dessous, le code de base du fichier App.svelte pour ce challenge :


            <script>
            // tous vos composants
            import jsPDF from "jspdf";
            import Header from "./components/Header.svelte";
            
            // Variables partagées
            const titre = "Un titre d'article Wikipédia";
            
            const generatePDF = () => {
                const doc = new jsPDF();
                // ici, vous construirez les pages du pdf.
                
                // Page 1
                doc.setFontSize(20);
                doc.text("Page 1", 10, 10);
                // Ajouter l'en-tête
                doc.text(titre, 10, 30);
                
                doc.save("example.pdf");
            }
            </script>
            
            <!-- Prévisualisation -->
            <div class="preview-container">
                <h2>Prévisualisation du PDF</h2>
            
                <!-- Page 1 -->
                <div class="page">
                    <Header {titre}/>
                </div>
            </div>

Règles pour l'élaboration du PDF

  • EN AUCUN CAS il ne vous est demandé de faire une interface graphique pour construire le PDF. Un bouton "Générer le PDF" suffira.
  • Comme pour le début du tp, il faut une fausse prévisualisation qui doit correspondre à peut près au pdf. Les composants vous serviront à créer cette fausse prévisualisation. Vous allez donc devoir coder une fois la fausse prévisualisation et une seconde fois pour le pdf. Ne vous inquiétez pas, avec les composants et l'export des variables, cela ne sera pas long.
  • Vous utiliserez les variables partagées dans App.svelte pour la création du pdf et la fausse prévisualisation.
  • Le PDF devra comporter au moins 2 pages.
  • Le code principal doit inclure au moins 6 composants, donc 6 fichiers différents (App.svelte exclu), parmi lesquels :
    • La date de création du PDF.
    • Une entête pour chaque page.
    • un pied de page pour chaque page.
    • Du texte comme contenu principal.
    • Au moins une image à mettre dans le dossier public
    • Le bouton pour générer le pdf
  • L’entête sera composé d’un titre d’un article quelconque trouvé sur Wikipédia.
  • Le pied de page sera composé des noms et prénoms du groupe.
  • Le contenu de ce tableau devra être affiché sur l'une des pages.
  • Vous êtes libre de choisir les textes à intégrer sur chaque page.
  • Le bouton se trouvera en bas de page en dessous de la prévisualisation.
  • Vous devez inclure la phrase "J'adore Svelte" sur chaque page.
  • Pour le bouton, celui-ci doit également être un composant et devra utiliser la directive d'événement on:click.
